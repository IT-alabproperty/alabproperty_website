# Обучение · ALAB Property как учебник

> Программа собрана под уже известный JS / CSS / HTML / Python / SQL.
> Стек ALAB: TypeScript · React 19 · Next.js 15 (App Router) · Tailwind · Supabase · Vercel.
> Цель: понимать каждую строку этого проекта и собрать аналог с нуля.
> Срок: **4–6 недель** при 5–10 ч/неделю → к лету готов к собственному проекту на любом стеке.

---

## Как пользоваться этим файлом

1. Иди по модулям сверху вниз — каждый = одно «блюдо» (концепция + теория + практика + тест).
2. **Не читай как книгу — пиши.** Каждый модуль заканчивается тестовой задачей в самом ALAB или соседнем мини-проекте. Без задачи модуль не считается пройденным.
3. После теста скидывай результат в чат с Claude (`@обучение.md модуль X — мой результат: ...`) → получаешь ревью.
4. Чекбоксы `[ ]` → `[x]` ниже — твой прогресс. Можно прямо в этом файле отмечать.

### Модуль 1 · TypeScript (1–2 дня) `[ ]`

Это твой JS + статические типы. После Python с type hints зайдёт легко.

**Что узнаешь:**
- `interface` vs `type` — когда что
- Generics: `Array<T>`, `Map<K, V>`, своя generic-функция
- Union types: `'sale' | 'rent'`, narrowing через type guards
- `Record<K, V>`, `Partial<T>`, `Omit<T, K>` — самые ходовые utility types
- `as`, `as const`, `satisfies` — три способа сужения типа

**Что читать:**
- [TypeScript Handbook · Everyday Types](https://www.typescriptlang.org/docs/handbook/2/everyday-types.html) — 30 минут
- [TS Cheatsheet React](https://react-typescript-cheatsheet.netlify.app/) — справочник, не читать целиком

**Где в ALAB:**
- [`lib/types.ts`](lib/types.ts) — типы домена (Property, Currency)
- [`lib/admin-auth.ts`](../admin-alabproperty/lib/admin-auth.ts) — generics в действии
- [`lib/pick.ts`](../admin-alabproperty/lib/pick.ts) — `pickAllowed<T>` пример

**Тест:**
1. Возьми любой `.js` файл из своего портфолио, переведи в `.ts` со строгим режимом
2. Опиши через `interface` структуру `Property` из ALAB **не подглядывая**. Сравни с реальной
3. Объясни без подглядывания: что делает `Partial<Property>` и где такое нужно

### Модуль 2 · React 19 + Server Components (2–3 дня) `[ ]`

**Главная новая тема.** Если ты раньше делал React — забудь старый mental model, в Next 15 другая логика.

**Что узнаешь:**
- Hooks: `useState`, `useEffect`, `useMemo`, `useRef`, `useCallback`, `useContext`

- Что такое hydration и почему её ломать нельзя
- `'use client'` директива — где ставить, где не ставить

**Что читать:**
- [React Quick Start](https://react.dev/learn) — официальные доки, 1-2 часа
- [Server Components](https://react.dev/reference/rsc/server-components) — отдельно
- [Next.js · Rendering](https://nextjs.org/docs/app/getting-started/server-and-client-components) — обязательно

**Где в ALAB:**
- [`app/layout.tsx`](app/layout.tsx) — server component
- [`components/property-card.tsx`](components/property-card.tsx) — client component, обрати внимание на `'use client'` сверху
- [`components/currency-context.tsx`](components/currency-context.tsx) — Context API
- [`components/scroll-reveal.tsx`](components/scroll-reveal.tsx) — `useEffect` + IntersectionObserver

**Тест:**
1. Объясни **построчно** что происходит в [`app/properties/[slug]/page.tsx`](app/properties/[slug]/page.tsx) — server или client, где async, где не async

К июлю когда летний проект с корешем — **80% знаний переносятся**:

| Концепция | Next.js (ALAB) | Альтернативы |

| Типизация | TypeScript | Python type hints / Rust types / Go types |
| Компоненты | React | Vue / Svelte / Solid |
| Routing | Next App Router | FastAPI / Express / Gin |
| ORM/Query | Supabase JS | SQLAlchemy / Prisma / GORM |
| RLS | PostgreSQL | **identical** |
| Auth | HMAC / cookies | **identical паттерны** |
| Deploy | Vercel | Railway / Fly.io / Render |
| SEO | Next metadata | **identical** требования |

Поменяется синтаксис фреймворка — пара дней на освоение. Концепции те же.
